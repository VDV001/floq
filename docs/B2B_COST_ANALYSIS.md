## Сценарий A: Внедрение для одного B2B-клиента (кастомный инструмент)

Floq в текущем виде — single-tenant MVP. Для одного клиента нужно минимум доработок.

### Разовые затраты

| Задача | Часы | Стоимость (3500 руб/ч) |
|--------|------|-----------------------|
| Деплой на VPS (Docker Compose) | 4-8 | 14-28K руб |
| Настройка домена, SSL, nginx reverse proxy | 2-4 | 7-14K руб |
| Настройка Telegram бота клиента | 1-2 | 3.5-7K руб |
| Настройка email (Resend + IMAP + SPF/DKIM) | 4-8 | 14-28K руб |
| Seed data, тестовый прогон, фикс багов | 8-16 | 28-56K руб |
| Обучение менеджеров (1-2 сессии) | 2-4 | 7-14K руб |
| **Итого разовых** | **21-42 ч** | **73-147K руб** |

### Ежемесячные затраты (инфраструктура)

| Статья | Стоимость/мес |
|--------|---------------|
| VPS (Hetzner CX32 или Selectel) | 1.5-3K руб |
| Домен | ~100 руб (1.2K руб/год) |
| AI API (Claude Sonnet, 200 лидов) | ~300-500 руб |
| AI API (Claude Sonnet, 1000 лидов) | ~1.5-2K руб |
| Resend (email отправка, до 3000/мес) | 0 руб (free tier) |
| Resend (до 50K/мес) | ~1.5K руб |
| **Итого/мес (малый объём)** | **2-4K руб** |
| **Итого/мес (средний объём)** | **5-7K руб** |

**Итого сценарий A: 75-150K руб разово + 2-7K руб/мес**

## Сценарий C: Ollama вместо Claude (экономия на AI)

Если клиент хочет zero AI cost — Floq уже поддерживает Ollama:

| Модель | Качество | Стоимость | Hardware |
|--------|----------|-----------|----------|
| Claude Sonnet | Отличное | ~$3-15/1M tokens | Облако |
| GPT-4o | Хорошее | ~$2.5-10/1M tokens | Облако |
| Llama 3.2 (Ollama) | Достаточное | **0 руб** | GPU на VPS (~5-10K руб/мес) |
| Mistral (Ollama) | Достаточное | **0 руб** | GPU на VPS |

С Ollama ежемесячные расходы на AI = 0, но нужен VPS с GPU (~5-10K руб/мес за T4). При большом объёме — выгоднее облачного API.

## Ключевые риски

- **Multi-tenancy** — самая трудозатратная доработка, без неё SaaS не запустить
- **Email deliverability** — репутация домена для cold outreach требует времени (2-4 недели warm-up)
- **AI-промпты на русском** — уже работают, но для каждой ниши клиента нужна тонкая настройка

## Ключевое преимущество

**MVP уже готов, архитектура чистая** (Clean Architecture + DDD), код масштабируется без переписывания. Это сильно снижает риск и время до рынка.
